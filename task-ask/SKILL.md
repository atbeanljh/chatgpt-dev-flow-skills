# 问答记录（Ask）

用于解答用户关于项目的各种问题，生成结构化的问答记录 `ask.md`。

与 task-workflow（需求→设计→计划→编码）不同，本 skill 是**独立的知识查询/解答流程**：
`理解问题 → 查阅代码 → 分析回答 → 记录归档`

## 工作流

### 1. 创建问答任务文件夹

运行 `scripts/init_task_folder.py <project-root> "<问题简述>"` 创建问答任务文件夹。
捕获输出的文件夹路径。

*注：如果暂时无法执行 init 脚本，也可手动在 `.codex/tasks/` 下创建文件夹。*

### 2. 理解问题

与用户确认需要解答的具体问题：
- 涉及哪个模块/功能？
- 想要了解什么方面的信息？
- 是否需要调研多种方案/对比？

### 3. 深入调研

查阅相关代码、文档、配置文件、调用链：
- 阅读相关模块源码
- 追踪数据流和调用链路
- 查看相关 API 定义和实现
- 查阅项目文档和注释
- 如需方案对比，列出各方案的优缺点

### 4. 编写 ask.md

使用下方文档结构写入任务文件夹的 `ask.md`。

### 5. 向用户交付

- 呈现问答记录给用户
- 确认是否解答了用户的问题
- 如有未覆盖的子问题，继续深入

## 文件结构

```
.codex/tasks/YYYY-MM-DD-<ask-title>/
  ├── ask.md          # 问答记录（核心文件）
  └── status.md       # 状态跟踪（由工作流维护）
```

## ask.md 文档结构

```markdown
# 问答记录

**创建时间**：YYYY-MM-DD HH:mm
**状态**：已答复
**问题**：<用户原始问题>
**涉及模块**：<模块/文件路径>

## 问题分析

<对问题的理解和拆解>

## 调研结果

### 1. <要点一>

<详细说明、代码示例、调用关系>

### 2. <要点二>

...

## 结论

<最终回答总结>

## 参考资料

- `<文件路径:行号>` — <说明>
- `<文件路径:行号>` — <说明>

## 扩展信息

<如果用户进一步追问，可以展开的内容>
```

## status.md 跟踪字段

| 字段 | 更新时机 | 示例值 |
|------|----------|--------|
| `ask_status` | 回答完成后 | ✅ 已完成 |
| `ask_artifact` | 回答完成后 | `ask.md` |
| `current_phase` | 状态变更 | 已交付 |
| `progress_note` | 每次关键操作 | 源码分析完成，待组织回答 |
| `todos` | 每次关键操作 | 1. 查阅 Service 层... |

## 中断与恢复

- 进度通过 `ask.md` 和 `status.md` 的 `progress_note` 记录
- 恢复时读取这两个文件，从断点继续

## 注意事项

- 引用代码时标注文件路径和行号
- 如有多种方案，要给出对比和推荐
- 对于不确定的信息，明确指出不确定性
- 回答可以包含代码示例、Mermaid 图表等增强理解
