# 计划阶段

基于已确认的 `02-design.md`，产出可执行任务清单 `tasks.md`。每个任务明确指定【目标对象】【修改目的】【修改方式】【修改内容】【相关依赖】。

这是**编码前的最后一个设计阶段**，输出结果直接作为 `task-code` 的实施依据。

本阶段是四个阶段的**第三步**：
`需求分析 → 方案设计 → 实施计划 → 编码实现`
由用户通过 `/code` 触发词手动调用下一阶段。

## 前置条件

- 任务文件夹中必须有已确认的 `02-design.md`（状态：已确认）
- 如果状态仍为"设计确认中"，不推进 — 先让用户确认
- 如果 status.md 中 plan_status 为"✅ 已完成"，询问是重新计划还是直接进入编码

## 工作流

### 0. 自动确认上一阶段

启动实施计划时，自动检查并确认上一阶段（方案设计）：
- 如果 `status.md` 中 `design_status` 不是"✅ 已完成"，自动将其更新为"✅ 已完成"
- 更新 `design_time` 为当前时间
- 在进度说明中记录"方案设计已自动确认"
- 向用户说明：方案设计阶段已完成、已自动确认，现进入实施计划

### 1. 定位任务文件夹

运行 `scripts/resolve_task_folder.py <project-root>` 或使用用户提供的路径。
读取 `status.md` 检查当前阶段和进度。

### 2. 读取文档

加载 `01-requirements.md` 和 `02-design.md`。

### 3. 分析代码库

确认每个【目标对象】文件路径存在。

### 4. 设计实现步骤

将方案拆分为可独立实施、可独立验证的任务：
- 任务间有清晰依赖关系
- 优先基础结构（数据库迁移、实体），再到业务逻辑，最后到接口/前端
- 每个任务标注【目标对象】【修改目的】【修改方式】【修改内容】

### 5. 编写 tasks.md

使用下方文档结构，写入 `03-tasks.md`。

### 6. 确认阶段完成

将文档给用户确认。
- **用户确认**：更新文档状态为"已确认"，更新 `status.md` 标记计划完成并切换到编码阶段，告知用户用 `/code` 进入编码。
- **用户要求修改**：按反馈修改，更新 `status.md` 进度说明，循环直到确认。

## 文档结构

```markdown
# 任务清单

**创建时间**：YYYY-MM-DD
**关联设计文档**：[02-design.md](./02-design.md)

---

## 整体依赖关系
<任务间的依赖图或文字说明>

---

- [ ] 1. <任务标题>
  - **变更ID**：<unique-id>
  - **依赖**：<前序任务ID或无>
  - **整体需求文档位置**：`requirements.md`
  - **整体设计文档路径**：`design.md`
  - **实现功能**：
    - <功能描述 bullet 1>
    - <功能描述 bullet 2>
  - **对应需求**：<需求编号或用户故事>
  - **目标对象**：
    - `<文件1路径>` — 修改/新增
    - `<文件2路径>` — 修改/新增
  - **修改目的**：<简要说明为什么要改>
  - **修改方式**：<如何修改的关键思路>
  - **修改内容**：
    - **步骤1**：<具体操作描述>
      ```<语言>
      // 代码/配置示意
      ```
    - **步骤2**：<具体操作描述>
  - **相关依赖**：<其他需要的类/方法/配置>

- [ ] 2. <任务标题>
  ...
```

> 注意：标记格式中的 `【目标对象】` 在 `.cospec/plan/changes/*/task.md` 中被大量使用，用于快速定位要修改的文件。在 `tasks.md` 中也建议使用这个标记来索引文件路径。

## status.md 更新说明

status.md 由工作流在执行过程中自动维护。

### 更新字段速查
| 字段 | 更新时机 | 示例值 |
|------|----------|--------|
| `plan_status` | 计划确认后 | ✅ 已完成 |
| `plan_time` | 计划确认后 | 2026-07-16 14:30 |
| `plan_artifact` | 计划确认后 | `03-tasks.md` |
| `current_phase` | 确认后切换 | 编码实现 |
| `progress_note` | 每次关键操作 | 计划完成，待用户确认 |
| `todos` | 每次关键操作 | 1. 确认任务清单... |

### 中断与恢复
- 进度通过 `progress_note` 和 `todos` 记录，阶段位置通过 `current_phase` 追踪
- 恢复时读取 status.md 跳过已完成阶段，从断点继续

## 查找已有任务文件夹

如果用户引用已有任务但没有提供路径，运行 `scripts/resolve_task_folder.py <project-root>` 选择最近的任务文件夹。
