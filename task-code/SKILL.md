# 编码实现阶段

基于已确认的 `03-tasks.md`，按任务清单逐一实现代码，编译验证，交付完成。

本阶段是四个阶段的**最后一步**：
`需求分析 → 方案设计 → 实施计划 → 编码实现`

## 前置条件

- 任务文件夹中必须有已确认的 `03-tasks.md`（状态：已确认）
- 对于简单任务，也可以直接从 `02-design.md` 开始实现（跳过 plan 步骤的简化流程）
- 如果 status.md 中 code_status 为"✅ 已完成"，直接询问用户是否需要重新编码或任务已经完成

## 工作流

### 0. 自动确认上一阶段

启动编码实现时，自动检查并确认上一阶段（实施计划）：
- 如果 `status.md` 中 `plan_status` 不是"✅ 已完成"，自动将其更新为"✅ 已完成"
- 更新 `plan_time` 为当前时间
- 在进度说明中记录"实施计划已自动确认"
- 向用户说明：实施计划阶段已完成、已自动确认，现进入编码实现

### 1. 定位任务文件夹

运行 `scripts/resolve_task_folder.py <project-root>` 或使用用户提供的路径。
读取 `status.md` 检查当前阶段和进度——确定已经完成到哪个任务了。

### 2. 读取所有文档

加载 `01-requirements.md`、`02-design.md`、`03-tasks.md` 中已存在的文件。

### 3. 分析代码库

确认代码库结构、构建配置、现有惯例。

### 4. 按任务清单实现

按 `03-tasks.md` 的顺序逐项实现：
- 每个任务完成后进行最小验证（编译通过）
- 前置任务不通过不进入下一个
- 每个任务完成后，将 `03-tasks.md` 中对应的 `[ ]` 更新为 `[x]`
- **同时更新 status.md**：记录当前完成到第几个任务、进度说明
- 如有冲突或歧义，停下来问用户
- **中断恢复时**：读取 tasks.md 中 `[x]` 标记和 status.md 进度，从第一个未完成的任务开始继续

### 5. 编译验证

运行 `mvn compile` 确认所有代码正确编译。

### 6. 运行测试

运行 `mvn test` 确认测试通过。

### 7. 打包产物

运行 `mvn package -DskipTests` 确认 JAR 包正常产出（仅 Maven 项目）。

### 8. 标记阶段完成

所有任务实现完毕后：
- 将 `03-tasks.md` 中所有 `[ ]` 更新为 `[x]`
- 更新 `status.md`：code_status → ✅ 已完成，code_time → 当前时间
- 确认验收标准满足情况
- 向用户交付：列出所有创建/修改的文件、总结实现内容、使用方式
- **自动进入收尾阶段**：编码交付后自动执行下方第 9 节的收尾工作

### 9. 自动收尾（最终清理）

所有编码任务和交付完成后，自动执行收尾工作：
- **确认任务清单**：最终确认 `03-tasks.md` 中所有任务项均已标记 `[x]`
- **更新状态文件**：更新 `status.md` 标记编码阶段完成、整个任务完成（code_status → ✅ 已完成，code_time → 当前时间）
- **检查任务文件夹**：检查并更新任务文件夹中的其他文件（如添加文件清单、更新已有文档的引用）
- **变更总结**：提供完整的变更总结——列出所有新增/修改/删除的文件、关键变更说明、使用方式
- **后续工作**：如果存在后续环节（如部署说明、发布检查、上线前注意事项），一并完成并告知用户
- **收尾确认**：确认所有文件已保存、状态已更新，向用户交付最终任务总结

## status.md 更新说明

status.md 由工作流在执行过程中自动维护。

### 更新字段速查
| 字段 | 更新时机 | 示例值 |
|------|----------|--------|
| `code_status` | 编码完成后 | ✅ 已完成 |
| `code_time` | 编码完成后 | 2026-07-16 14:30 |
| `code_artifact` | 编码完成后 | 代码文件清单 |
| `cleanup_status` | 收尾完成后 | ✅ 已完成 |
| `cleanup_time` | 收尾完成后 | 2026-07-16 15:00 |
| `progress_note` | 每个任务完成后 | 已完成任务1-3，当前进行任务4 |
| `todos` | 每个任务完成后 | 1. 实现任务4... |

### 中断与恢复
- 任务级进度：通过 `03-tasks.md` 中的 `[x]` 标记和 `status.md` 的 `progress_note` 双重记录
- 恢复时读取 tasks.md，找到第一个 `[ ]` 任务开始
- 已完成的任务跳过，直接从断点继续编码

## 注意事项

- 遵循 `03-tasks.md` 中的【目标对象】【修改目的】【修改方式】【修改内容】标记进行实现
- 每个任务独立实现，完一个再进下一个
- **中断后恢复**：读取 status.md 获得上下文概述，然后直接读取 tasks.md 找到断点
- 每次大块工作后（如完成一个任务），更新 status.md 的 progress_note
- 遇到设计文档中没有覆盖的细节，用最佳实践补充并用注释说明

## 资源
